## Придумать схему кодирования

Необходимо разработать схему кодирования для алфавита, состоящего из следующих символов:

- кириллические символы верхнего и нижнего регистра (включая букву «ё»);
- цифры;
- знаки препинания;
- управляющие символы (пробел, табуляция, перевод строки).

**Главное требование:** схема должна быть максимально экономичной по памяти.

**Подсказка:** необязательно кодировать все символы «как есть», как и необязательно делать фиксированный размер на символ.

### Простой пример

Допустим, я придумываю кодовую таблицу для кодирования трёх букв: a, b, c.

Для их кодирования хватит всего двух бит:

- a — 00
- b — 01
- c — 10

Тогда в одном байте можно передать четыре таких символа!

Но что, если я хочу ещё уметь передавать эти же символы, но в верхнем регистре? Тогда получается, что нужно уже три бита на символ:

- a — 000
- b — 001
- c — 010
- A — 011
- B — 100
- C — 101

Или можно пойти на хитрость и не кодировать отдельно символы в верхнем регистре, а сделать модификатор смены регистра. Тогда можно остаться с двумя битами:

- a — 00
- b — 01
- c — 10
- CAPS — 11

Можно ввести правило: если в сообщении встречается CAPS, то все последующие символы должны быть в альтернативном регистре, пока снова не появится CAPS.

Разумеется, у такого подхода есть и минусы: например, для ситуаций, когда только один символ в верхнем регистре, потребуется уже шесть бит, а не три...

Одним словом, задачка на подумать, и правильного решения тут нет. Её смысл — чтобы вы поняли правила игры с битами и кодированием.

| В      | 2              | 00000010     |
| Г      | 3              | 00000011     |
| ...    | ...            | ...          |
| я      | 65             | 01000001     |
| 0      | 66             | 01000010     |
| 1      | 67             | 01000011     |
| ...    | ...            | ...          |
| №      | 94             | 01011110     |
