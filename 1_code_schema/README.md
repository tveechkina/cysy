```
Необходимо разработать схему кодирования для алфавита, состоящего из следующих символов:
- кириллические символы верхнего и нижнего регистра (включая букву «ё»);
- цифры;
- знаки препинания;
- управляющие символы (пробел, табуляция, перевод строки).

Главное требование: схема должна быть максимально экономичной по памяти.

Подсказка: необязательно кодировать все символы «как есть», как и необязательно делать фиксированный размер на символ.
```

Исходя из главного требования к кодированию, было решено сделать приоритетом экономию памяти, а не расширяемость схемы. Для этого использовался алгоритм Хаффмана и переменное кол-во битов на символ алфавита.

Для кодирования алфавита необходимо:

1.  Определить частотность каждого символа алфавита в рамках текстового корпуса.
2.  Исключить верхний регистр букв и наиболее редко используемые символы пунктуации. Они будут кодироваться отдельно с использованием служебных команд.
3.  Включить в таблицу частотности служебные команды SHIFT (условная частотность 5%) и CAPS (0,3%).
4.  Составить частотное дерево по Хаффману. Алгоритм:

    4.1 В отсортированной таблице частотности сравниваем два самых маленьких показателя.

    4.2 Удаляем эти два значения из списка и добавляем вместо них новый узел, равный их сумме.

    4.3 Продолжаем сравнивать минимальные показатели частотности до тех пор пока не сформируется корневой узел, равный сумме всех частот.

    Например, для таблицы кириллических букв дерево Хаффмана будет выглядеть следующим образом:

     <div style="display: flex;">
     <div style="flex: 1;">

    | Буква | %   |
    | ----- | --- |
    | ю     | 0.4 |
    | ц     | 0.3 |
    | э     | 0.2 |
    | щ     | 0.2 |
    | ф     | 0.2 |
    | ё     | 0.1 |

     </div>
     <div style="flex: 2;">

            (... продолжать до корня)
                     /      \
                (0.6)         (0.8)
                /  \          /  \
            (0.3)  (0.3)  (0.4)  (0.4)
            /  \     ц     /  \    ю
         (0.1) (0.2)   (0.2)  (0.2)
           ё     ф       щ      э

     </div>
     </div>

5.  После построения дерева каждому переходу по ветви назначается бит:

- левая ветвь — `0`;
- правая ветвь — `1`;

  Так, для куска дерева, представленного выше, кодировка выглядела бы так:

                   [...КОРЕНЬ...]
                       /      \
                  (0.6)        (0.8)
                   [0]          [1]
                  /  \          /  \
              (0.3)  (0.3)  (0.4)  (0.4)
               [0]    [1]    [0]    [1]
              /  \     ц     /  \    ю
           (0.1) (0.2)   (0.2)  (0.2)
            [0]   [1]     [0]    [1]
             ё     ф       щ      э

Значит, символы будут соответствовать таким кодам:
| Буква | Биты|
| ----- | --- |
| ю | 11 |
| ц | 01 |
| э | 101 |
| щ | 100 |
| ф | 001 |
| ё | 000 |

При формировании дерева для всего алфавита будет видно, что для символов с наименьшей частотой используется большее кол-во бит. При этом часто встречающиеся символы кодируются меньшим количеством бит.

6. В сформированном дереве Хаффмана будут закодированы служебные команды `SHIFT` (смена регистра на одну букву) и `CAPS` (смена регистра всех последующих букв).

   Некоторые редкие знаки препинания не включаются в основное дерево, а кодируются как последовательность из служебной команды `SHIFT` и базового символа.

   Например, символ `:` может кодироваться как последовательность `SHIFT` и базового символа `0`.
